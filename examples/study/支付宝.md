## 一、前置准备（必须先做）

### 第1步：注册支付宝开放平台账号

1. 打开 https://open.alipay.com
2. 用企业支付宝账号登录（**个人账号有限制，建议用企业账号**）
3. 点击右上角「控制台」进入开发者后台

> **没有企业账号？** 先用沙盒环境测试，沙盒不需要企业资质，个人就能用。

---

### 第2步：创建应用

1. 进入控制台 → 点击「创建应用」
2. 选择「网页&移动应用」
3. 填写应用名称，上传图标
4. 提交后等待审核（沙盒不需要审核）

创建成功后，你会得到一个 **APPID**，类似 `2021001234567890`，记下来。

---

### 第3步：配置密钥（最重要的步骤）

支付宝使用「公钥证书模式」，你需要生成一对密钥，然后上传公钥给支付宝，支付宝返回 3 个证书文件给你。

#### 3.1 下载密钥生成工具

前往支付宝文档页下载「支付宝开放平台开发助手」：
- Windows: https://opendocs.alipay.com/common/02khjo（官方工具页面搜索"开发助手"）

#### 3.2 生成密钥对

1. 打开开发助手
2. 点击「生成密钥」
3. 密钥格式选 **PKCS1**
4. 密钥长度选 **2048**
5. 点击「生成密钥」按钮
6. 保存两个文件（**私钥绝对不能泄露！**）：
   - `应用私钥` → 复制保存到本地文件 `private_key.pem`
   - `应用公钥` → 稍后上传给支付宝

#### 3.3 上传公钥，下载证书

1. 进入支付宝控制台 → 你的应用 → 「开发设置」
2. 加密方式选「公钥证书」
3. 点击「上传应用公钥证书」，把 **应用公钥** 内容粘贴进去
4. 上传成功后，页面会出现 3 个可下载的文件：

| 文件名 | 说明 | 配置项 |
|--------|------|--------|
| `appCertPublicKey_xxxx.crt` | 应用公钥证书 | `app_public_cert_path` |
| `alipayCertPublicKey_RSA2.crt` | 支付宝公钥证书 | `alipay_public_cert_path` |
| `alipayRootCert.crt` | 支付宝根证书 | `alipay_root_cert_path` |

**把这 3 个文件下载到你项目的 `certs/` 目录里，妥善保管。**

#### 3.4 整理好文件结构

```
你的项目/
├── certs/
│   ├── private_key.pem               ← 你的应用私钥
│   ├── appCertPublicKey_xxxx.crt     ← 应用公钥证书
│   ├── alipayCertPublicKey_RSA2.crt  ← 支付宝公钥证书
│   └── alipayRootCert.crt            ← 支付宝根证书
└── your_code.py
```

### 第4步：配置回调地址（notify_url）

支付宝支付完成后，会向你的服务器发送一个 POST 请求通知你（这叫"异步回调"）。

**要求：**
- 必须是**公网可访问**的地址（localhost 不行）
- 必须是 http/https，端口 80 或 443
- 格式例如：`https://你的域名.com/notify/alipay`

> **本地开发怎么办？** 使用内网穿透工具，如 ngrok、frp、cpolar。  
> 例如用 ngrok：`ngrok http 8000`，它会给你一个公网地址。

### 4.2 网页支付（PC 浏览器，最常用）

用户在 PC 浏览器打开支付宝收银台页面完成支付。

```python
from senweaver_pay import Pay
from senweaver_pay.types import PaymentRequest, PaymentChannel, PaymentMethod, UnifiedOrder
from decimal import Decimal

# 构造订单
order = UnifiedOrder(
    out_trade_no="ORDER_20250605_001",  # 你系统里的唯一订单号
    subject="商品名称",                  # 显示给用户看的标题
    body="商品详情描述",                 # 可选
    total_amount=Decimal("0.01"),       # 金额，单位：元
    notify_url="https://你的域名.com/notify/alipay",   # 回调地址
    return_url="https://你的域名.com/return/alipay",   # 支付完成跳转地址
)

# 发起支付
request = PaymentRequest(
    channel=PaymentChannel.ALIPAY,
    method=PaymentMethod.WEB,    # 网页支付
    order=order,
)

response = Pay.create(request)

if response.success:
    # form_data 是一段 HTML 表单，直接返回给前端展示即可
    # 浏览器会自动跳转到支付宝收银台
    html_form = response.form_data
    print("支付表单生成成功，返回给前端")
    # 在 Flask/FastAPI 中：return HTMLResponse(content=html_form)
else:
    print(f"支付失败: {response.message}")
```

**前端处理：** 将 `response.form_data` 直接作为 HTML 返回，浏览器会自动 POST 跳转到支付宝。

## 五、写代码：处理回调通知

> 支付成功后，支付宝会向你的 `notify_url` 发送 POST 请求。  
> **你必须处理这个请求，然后回复 `success` 字符串，否则支付宝会重复通知。**

### Flask 示例

```python
from flask import Flask, request
from senweaver_pay import Pay

app = Flask(__name__)

@app.route("/notify/alipay", methods=["POST"])
def alipay_notify():
    # 获取支付宝 POST 过来的表单数据
    form_data = request.form.to_dict()

    try:
        alipay = Pay.get_channel("alipay")
        result = alipay.callback(form_data=form_data)

        if result.success:
            # 验签成功，处理你的业务逻辑
            order_no = result.out_trade_no    # 你的订单号
            trade_no = result.trade_no        # 支付宝流水号
            amount = result.amount            # 支付金额
            status = result.status            # 支付状态

            print(f"订单 {order_no} 支付成功，金额 {amount} 元")

            # ★ 在这里更新你的数据库订单状态 ★
            # update_order_status(order_no, "paid")

            # 必须返回 "success"，否则支付宝会持续重试通知
            return alipay.success()
        else:
            return "fail"

    except Exception as e:
        print(f"回调处理异常: {e}")
        return "fail"
```

### FastAPI 示例

```python
from fastapi import FastAPI, Request
from fastapi.responses import PlainTextResponse
from senweaver_pay import Pay

app = FastAPI()

@app.post("/notify/alipay")
async def alipay_notify(request: Request):
    form_data = dict(await request.form())

    try:
        alipay = Pay.get_channel("alipay")
        result = alipay.callback(form_data=form_data)

        if result.success:
            order_no = result.out_trade_no
            print(f"订单 {order_no} 支付成功")

            # ★ 更新数据库 ★

            return PlainTextResponse("success")
        else:
            return PlainTextResponse("fail")

    except Exception as e:
        print(f"异常: {e}")
        return PlainTextResponse("fail")
```

### 回调数据字段说明

| 字段 | 说明 |
|------|------|
| `result.out_trade_no` | 你系统的订单号 |
| `result.trade_no` | 支付宝交易流水号 |
| `result.amount` | 支付金额（字符串，如 "0.01"） |
| `result.pay_time` | 支付时间（如 "2025-06-05 10:30:00"） |
| `result.status` | 支付状态枚举 |

### 支付状态枚举说明

```python
from senweaver_pay.types import PaymentStatus

PaymentStatus.SUCCESS    # 支付成功
PaymentStatus.PENDING    # 等待支付
PaymentStatus.CANCELLED  # 已取消
PaymentStatus.CLOSED     # 已关闭
PaymentStatus.FAILED     # 支付失败
```

## 六、写代码：查询订单

用于主动查询订单当前状态（比如回调没收到时用来兜底）。

```python
from senweaver_pay import Pay
from senweaver_pay.types import QueryRequest, PaymentChannel

query_request = QueryRequest(
    channel=PaymentChannel.ALIPAY,
    out_trade_no="ORDER_20250605_001",  # 你的订单号
)

result = Pay.query(query_request)

if result.success:
    print(f"订单状态: {result.status}")
    print(f"支付金额: {result.amount}")
    print(f"支付时间: {result.pay_time}")
    print(f"买家账号: {result.buyer_logon_id}")
else:
    print(f"查询失败: {result.message}")
```

## 七、写代码：申请退款

```python
from senweaver_pay import Pay
from senweaver_pay.types import RefundRequest, PaymentChannel
from decimal import Decimal

refund_request = RefundRequest(
    channel=PaymentChannel.ALIPAY,
    out_trade_no="ORDER_20250605_001",    # 原订单号
    out_refund_no="REFUND_20250605_001",  # 你系统生成的退款单号（唯一）
    refund_amount=Decimal("0.01"),        # 退款金额，不能超过原订单金额
    refund_reason="用户申请退款",          # 退款原因（可选）
)

result = Pay.refund(refund_request)

if result.success:
    print(f"退款成功！退款金额: {result.refund_amount} 元")
    print(f"退款时间: {result.refund_time}")
else:
    print(f"退款失败: {result.message}")
```

**注意：**
- 支付宝退款**不需要**用户操作，直接原路退回
- 同一笔订单可多次部分退款，但总退款金额不能超过订单金额
- 退款到账时间：实时到账（通常几秒内）

## 九、常见错误排查

### 错误 1：证书文件不存在

```
InvalidConfigException: 应用公钥证书文件不存在: certs/appCertPublicKey.crt
```

**解决：** 检查 `app_public_cert_path` 路径是否正确，使用绝对路径更可靠：

```python
import os
BASE_DIR = os.path.dirname(os.path.abspath(__file__))

config = {
    "alipay": {
        "default": {
            "app_public_cert_path": os.path.join(BASE_DIR, "certs", "appCertPublicKey_xxxx.crt"),
            "alipay_public_cert_path": os.path.join(BASE_DIR, "certs", "alipayCertPublicKey_RSA2.crt"),
            "alipay_root_cert_path": os.path.join(BASE_DIR, "certs", "alipayRootCert.crt"),
            # ...
        }
    }
}
```

---

### 错误 2：签名验证失败

```
InvalidSignException: Invalid signature in response
```

**常见原因：**

| 原因 | 解决方式 |
|------|----------|
| 私钥和公钥不是一对 | 重新生成密钥对，重新上传公钥 |
| 使用了错误环境的证书 | 沙盒用沙盒证书，正式用正式证书 |
| `app_id` 填错 | 检查 `app_id` 和证书是否匹配 |
| 证书文件内容损坏 | 重新下载证书 |

---

### 错误 3：app_id 不存在或未开通权限

```
GatewayException: 支付宝API错误: 应用公钥证书不存在
```

**解决：**
1. 登录支付宝控制台，确认应用状态是「上线」而非「开发中」
2. 确认已开通「电脑网站支付」或对应支付产品的权限
3. 沙盒环境确认使用的是沙盒 `app_id`

---

### 错误 4：notify_url 收不到回调

**检查清单：**
- [ ] `notify_url` 是否是公网可访问的地址（不能是 localhost）
- [ ] 服务器防火墙是否开放 80/443 端口
- [ ] URL 路径是否正确（大小写敏感）
- [ ] 接口是否返回了字符串 `"success"`（不是 JSON，不是 HTTP 200）

**调试技巧：** 先用 https://webhook.site 创建一个临时接收地址，验证支付宝确实在发通知。

---

### 错误 5：私钥格式错误

```
InvalidConfigException: Failed to load private key
```

私钥文件内容必须包含完整的 PEM 头尾：

```
-----BEGIN RSA PRIVATE KEY-----
MIIEow...（很长的一串）
-----END RSA PRIVATE KEY-----
```

如果用环境变量存储私钥，换行符要用 `\n`：

```python
import os
private_key = os.environ.get("ALIPAY_PRIVATE_KEY").replace("\\n", "\n")
```

## 十、正式上线检查清单

上线前逐项确认：

- [ ] 将 `mode` 从 `"sandbox"` 改为 `"normal"`
- [ ] 替换为正式环境的 APPID（不是沙盒 APPID）
- [ ] 替换为正式环境的证书文件（重新生成密钥对，重新下载 3 个证书）
- [ ] `notify_url` 改为正式域名的地址
- [ ] `return_url` 改为正式域名的地址
- [ ] 证书文件权限设置为仅服务进程可读（避免泄露私钥）
- [ ] 私钥不要提交到 Git（在 `.gitignore` 里加上 `certs/`）
- [ ] 回调接口做了幂等处理（同一订单多次回调只处理一次）
- [ ] 在支付宝控制台开通了对应支付产品的权限

## 附：完整流程图

```
用户点击「去支付」
       ↓
你的服务器调用 Pay.create()
       ↓
返回 HTML 表单 (form_data) 给前端
       ↓
浏览器自动跳转到支付宝收银台
       ↓
用户完成支付
       ↓
支付宝异步 POST 你的 notify_url  ← 最重要，在这里更新订单状态
       ↓
你的服务器返回字符串 "success"
       ↓
支付宝同步跳转用户浏览器到 return_url（展示支付成功页面）
```

> **提示：** 不要依赖 `return_url` 来判断支付是否成功，应该以 `notify_url` 回调为准。  
> 因为用户可能关闭浏览器导致 `return_url` 没有被访问到，但 `notify_url` 回调是服务器对服务器的，更可靠。
